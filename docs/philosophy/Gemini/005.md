# Gemini Response 005: Flame Energy Inside The Graph

Date: 2026-06-03

Source: 사용자가 Gemini에게 "문제 풀이 중 동적 불꽃 로고" 아이디어를 묻고 받은 답변을 SKAI 실행 문서로 정리한 것이다.

## Core Decision

Literal flame icon should not become SKAI's primary brand mark.

The primary branding direction remains the 3-node directed graph:

```text
Human Intent / Framing
+ Messy Reality / Materials
-> Synthesized Artifact
```

Reason:

- Flame emphasizes AI's power, energy, and mythic capability.
- Graph emphasizes structure, architecture, control, and user agency.
- SKAI's core message is not "AI is powerful"; it is "the user can structure intelligence."

## Prometheus Constraint

Prometheus/fire is still a valid origin metaphor:

- fire = capability,
- wood = framing,
- hearth = workflow,
- safety = verification.

But the UI should not drift into mythic, heroic, or theatrical AI branding. If the logo becomes a flame, users may read SKAI as another AI wrapper or prompt trick product. The graph mark keeps the user in the role of orchestrator.

## Accepted Fusion

The founder's desire for "dynamic energy while solving" is valuable. The right implementation is not a flame shape, but flame-like energy contained inside the graph structure.

Engine Mode dynamic mark:

1. Idle/ready state:
   - The 3-node hex graph is precise and quiet.

2. Primed state:
   - When the user attaches material or writes a prompt, intent/material packets become more visible.

3. Busy state:
   - When the backend/model is processing, packets move faster from intent/material nodes toward artifact.
   - The motion can feel like sparks or current, but it remains graph flow.

4. Structured state:
   - After trace events accumulate, the artifact node pulses subtly.
   - This communicates that the user's dialogue has become a structure.

## Product Meaning

This preserves the core philosophy:

- no literal flame-first brand,
- no AI worship,
- no wrapper-like visual trope,
- no prompt-trick theatrics.

But it gives Engine Mode a stronger sensory feedback loop:

- the user sees that attaching materials and sending prompts "feeds" the graph,
- the output node becomes active as the artifact forms,
- the mark feels alive without abandoning structure.

## Implementation Direction

Add an activity-aware packet-flow layer:

- use `html[data-skai-activity]` or equivalent root state,
- keep default Engine Mode packet motion subtle,
- increase speed/density in busy state,
- add no flame SVG shape,
- keep reduced-motion fallback,
- document that this is not brand replacement but Engine Mode activity feedback.

## Philosophy Check

This response is accepted. It strengthens SKAI's visual energy while protecting the 3-node graph as the primary identity and system metaphor.
