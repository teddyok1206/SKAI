# SKAI Foundation Questions

이 문서는 SKAI의 철학, 방향성, 평가관, 커뮤니티 구조를 결정하기 위한 질문 목록이다. 지금 당장 모두 답할 필요는 없지만, 답하지 않은 질문은 제품 판단의 빚으로 남는다.

## 0. 정체성

1. SKAI의 공식 풀네임은 `Social Knowledge of AI`인가, 아니면 현재 표기인 `Social Knowlege of AI`를 의도적으로 유지할 것인가?
2. SKAI는 교육 플랫폼인가, 평가 플랫폼인가, 커뮤니티인가, 모델 벤치마크 플랫폼인가, 채용/인증 도구인가?
3. 첫 데모에서 가장 먼저 증명해야 하는 명제는 무엇인가?
4. SKAI가 절대 포기하면 안 되는 핵심 철학 한 문장은 무엇인가?
5. SKAI가 기존 프롬프트 강의, 유튜브 튜토리얼, 프롬프트 모음집과 근본적으로 다른 지점은 무엇인가?
6. SKAI의 경쟁자는 백준/프로그래머스에 가까운가, Kaggle에 가까운가, Hugging Face에 가까운가, 사내 AX 교육 도구에 가까운가?
7. SKAI는 "AI를 잘 쓰는 사람"을 길러내는가, "AI와 함께 문제를 해결하는 구조를 설계하는 사람"을 길러내는가?

## 1. 평가하려는 능력

1. SKAI가 평가하는 역량의 최소 단위는 무엇인가: 프롬프트 하나, 대화 전체, 문제 해결 전략, 최종 산출물, 혹은 사용자-모델 조합?
2. 문제 정의 능력, 세분화 능력, 프롬프트 작성 능력, 모델 선택 능력, 검증 능력, 비용 관리 능력 중 무엇이 핵심인가?
3. "좋은 AI 오케스트레이션"은 빠른가, 정확한가, 저렴한가, 재현 가능한가, 학습 가능한가, 설명 가능한가?
4. 사용자가 AI에게 바로 답을 요구하지 않고 먼저 문제를 구조화하면 점수를 더 받아야 하는가?
5. 사용자가 스스로 도메인 지식을 투입한 경우, 그 지식이 맞는지 틀리는지 어떻게 평가할 것인가?
6. 사용자가 AI의 잘못된 답을 감지하고 수정한 행동은 어떤 점수로 반영할 것인가?
7. "모르는 것을 모른다고 판단하고 추가 정보를 요청하는 능력"을 어떻게 보상할 것인가?
8. 문제 정의가 흔들렸지만 최종 산출물은 좋은 경우, 점수는 어떻게 줘야 하는가?
9. 최종 산출물은 평범하지만 과정이 매우 모범적인 경우, 점수는 어떻게 줘야 하는가?
10. 사용자가 모델의 한계를 우회한 능력과 모델 자체의 성능을 어떻게 분리할 것인가?

## 2. 문제 철학

1. SKAI 문제는 정답이 있는 문제인가, 좋은 해결안이 여러 개인 문제인가?
2. 문제는 얼마나 불명확해야 하는가?
3. 문제 설명에 일부러 누락된 조건을 넣을 것인가?
4. 사용자가 clarifying question을 던질 수 있는가?
5. 문제 속 "의뢰자", "고객", "상사", "데이터 제공자" 같은 역할을 시뮬레이션할 것인가?
6. 문제는 현실 업무형, 연구형, 창작형, 데이터 분석형, 코딩형, 자료조사형 중 어디서 시작할 것인가?
7. 초보자용 문제와 전문가용 문제는 같은 평가 체계를 쓸 것인가?
8. 하나의 문제에 대해 공식 모범 풀이가 있어야 하는가?
9. 문제별로 숨겨진 채점 기준을 둘 것인가, 아니면 모든 기준을 공개할 것인가?
10. 백준처럼 티어를 둘 경우, 난이도는 무엇으로 결정할 것인가?

## 3. 피드백 철학

1. 사용자에게 가장 가치 있는 피드백은 점수인가, 과정 분석인가, 다음 개선 프롬프트인가, 다른 사람의 풀이 비교인가?
2. bottleneck prompt를 어떻게 정의할 것인가?
3. "너무 추상적인 질문", "문제 정의 이탈", "불필요한 context reload", "검증 부재" 같은 실패 유형을 표준 taxonomy로 만들 것인가?
4. 피드백은 엄격한 채점표처럼 보여줄 것인가, 코치의 리뷰처럼 보여줄 것인가?
5. 사용자의 자존심을 덜 건드리면서도 날카로운 피드백을 주는 톤은 무엇인가?
6. 피드백이 틀렸을 때 사용자는 이의를 제기할 수 있어야 하는가?
7. judge가 지적한 문제를 사용자가 수정해 재제출할 수 있는가?
8. 한 시도 안에서 "좋았던 선택"과 "나빴던 선택"을 모두 보여줄 것인가?
9. 초보자에게는 무엇을 감추고 무엇을 보여줘야 하는가?
10. 전문가에게는 raw trace와 judge rationale을 어디까지 공개할 것인가?

## 4. 점수와 정량화

1. 총점은 하나로 합칠 것인가, 아니면 여러 축의 radar score로 보여줄 것인가?
2. 정량화할 수 있는 지표는 무엇인가: 토큰, 비용, latency, turn count, context switching, schema validity, clarification count, retry count?
3. 정량화하지 말아야 할 지표는 무엇인가?
4. LLM judge 점수와 deterministic metric의 비율은 어떻게 둘 것인가?
5. 모델별 점수를 분리할 것인가?
6. 문제별 리더보드는 모델 성능 차이를 어떻게 보정할 것인가?
7. 같은 사용자가 같은 문제를 반복해서 풀면 점수와 랭킹은 어떻게 처리할 것인가?
8. 점수 조작을 막기 위해 무엇을 제한할 것인가?
9. 최종 점수는 인증/채용에 쓸 수 있을 정도의 신뢰도를 목표로 할 것인가?
10. "좋은 질문을 던졌지만 답을 완성하지 못한 시도"는 어떻게 기록할 것인가?

## 5. 공개와 커뮤니티

1. 다른 사람의 풀이에서 무엇을 볼 수 있어야 하는가: 프롬프트 순서, 전체 프롬프트, 모델 응답, judge 리뷰, 최종 산출물?
2. 사용자가 프롬프트를 공개하면 어떤 이익을 얻는가?
3. 비공개 풀이와 공개 풀이를 구분할 것인가?
4. 공개된 프롬프트가 곧바로 복붙되는 문제를 어떻게 볼 것인가?
5. 공개 prompt trace의 저작권과 라이선스는 누구에게 있는가?
6. 댓글, 리뷰, fork, remix 같은 GitHub식 상호작용을 넣을 것인가?
7. 좋은 풀이를 큐레이션하는 운영자/모더레이터가 필요한가?
8. 커뮤니티의 핵심 문화는 경쟁인가, 학습인가, 연구인가, 포트폴리오인가?
9. 악의적인 프롬프트, 개인정보, 회사 기밀이 들어온 경우 어떻게 처리할 것인가?
10. 글로벌 커뮤니티를 목표로 할 때 한국어 풀이와 영어 풀이를 어떻게 연결할 것인가?

## 6. 사용자 세그먼트

1. 첫 사용자는 회사 AX 세미나 참가자인가, 대학생인가, 개발자인가, 일반인인가?
2. "AI 문외한"을 첫 데모의 핵심 사용자로 둘 것인가, 아니면 두 번째 단계로 둘 것인가?
3. 개발자에게 제공해야 할 가치는 무엇인가?
4. 기업 실무자에게 제공해야 할 가치는 무엇인가?
5. 기업 인사팀에게 제공해야 할 가치는 무엇인가?
6. 모델 스타트업에게 제공해야 할 가치는 무엇인가?
7. 빅테크나 모델 제공사가 얻는 데이터/피드백은 무엇인가?
8. 각 세그먼트가 서로 충돌할 때 누구를 우선할 것인가?
9. 첫 10명의 사용자를 어디서 데려올 것인가?
10. 첫 100명의 사용자가 반복 방문할 이유는 무엇인가?

## 7. 수익과 인센티브

1. 데모 단계에서 수익화는 완전히 배제할 것인가?
2. 사용자가 직접 API key를 가져오는 모델을 허용할 것인가?
3. 모델 스타트업이 서버/모델을 제공하고 SKAI가 정제된 사용 데이터를 제공하는 구조를 언제부터 시도할 것인가?
4. 사용 데이터 제공은 opt-in인가, 기본값인가?
5. 데이터 제공 대가로 사용자는 무료 크레딧, 뱃지, 랭킹, 모델 접근권 중 무엇을 받는가?
6. 기업 교육용 유료 세션과 공개 커뮤니티는 어떻게 분리할 것인가?
7. 인증서나 점수 리포트를 유료화할 수 있는가?
8. 문제 출제자, 풀이 공개자, 리뷰어에게 보상 구조를 줄 것인가?
9. 채용/평가 시장으로 가기 전 필요한 신뢰도 기준은 무엇인가?
10. 수익화가 교육/커뮤니티 철학을 훼손하지 않게 하는 금지선은 무엇인가?

## 8. 윤리와 신뢰

1. 사용자의 prompt trace는 개인정보인가, 학습 데이터인가, 포트폴리오인가?
2. 회사 업무 데이터나 고객 데이터가 들어오면 어떻게 막거나 보호할 것인가?
3. judge가 편향된 피드백을 할 때 어떻게 감지할 것인가?
4. 모델 제공사에게 데이터를 줄 때 익명화 기준은 무엇인가?
5. 미성년자나 학생 사용자의 데이터를 어떻게 다룰 것인가?
6. 공개 리더보드가 초보자의 학습 의욕을 꺾을 가능성은 없는가?
7. AI 활용 능력을 점수화하는 것이 불공정하게 쓰일 위험은 무엇인가?
8. 기업이 SKAI 점수를 채용에서 사용할 때 필요한 고지와 제한은 무엇인가?
9. prompt trace 공개가 개인의 사고 과정을 과도하게 노출하는 문제는 없는가?
10. SKAI가 "AI를 잘 쓰는 법"을 넘어 "AI에 과도하게 의존하는 법"을 가르치지 않으려면 무엇이 필요한가?

## 9. 첫 데모 성공 기준

1. AX 세미나에서 데모가 성공했다고 말할 수 있는 최소 결과는 무엇인가?
2. 참가자는 몇 명인가?
3. 세미나 시간은 몇 분인가?
4. 참가자들은 개인 노트북을 쓰는가, 휴대폰을 쓰는가?
5. 참가자들이 OpenAI/Gemini/Hugging Face 계정을 직접 만들어야 하는가?
6. 데모 당일 반드시 보여줘야 하는 화면은 무엇인가?
7. 데모 당일 반드시 수집해야 하는 데이터는 무엇인가?
8. 데모 직후 참가자에게 어떤 리포트를 줄 것인가?
9. 데모 후 대표에게 보여줄 수 있는 성과 지표는 무엇인가?
10. 첫 데모에서 실패해도 괜찮은 부분과 실패하면 안 되는 부분은 무엇인가?
