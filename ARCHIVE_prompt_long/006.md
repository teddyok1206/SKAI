지금 그 프롬프트 입력하는 곳에 파일 넣는 곳에도 드래그가 되고 텍스트 치는 곳에도 드래그가 되는데 파일 넣는 곳에만 드래그 되도록 해줘. 그리고 내가 생각한 data structure 아이디어인데, 이용자의 프롬프트 집합과 AI의 응답 집합을 directed 2 dim dual graph로 관리하고 하나의 프롬프트 & 응답 쌍마다 task status를 dual로 매칭시켜서 3 dim directed graph를 만드는거 어때? 그러면 프롬프트 목록 보여줄 때 각 노드는 이용자의 프롬프트가 되고 그 사이의 directed edge가 LLM의 응답이 되면서 동시에 이건 응답 그래프의 노드가 되고, AI 그래프 입장에서의 edge가 사용자의 프롬프트인데 이건 노드의 input이 엣지를 통해 들어오고 그에 대한 모델 응답이 노드가 되면서 아주 자연스러운 흐름이 되잖아. 그래프가 있으면 시각화 하기도 좋을 거고, 자료구조 측면에서 incidence matrix, 딕셔너리 등으로 관리하면 되니까 훨씬 빠르고 편리할 거 같아. 번외로 프롬프트 엔지니어링 분야에서 이 그래프들과 채점 결과를 매칭시켜서 공학적 연구를 할 수도 있겠지. 매우 중요한 아이디어니까, 진짜 절대 구현 안 되는 정도만 아니면 문서에 깊숙히 박아두고 실제로 구현할 수 있는 건 최대한 구현해줘
