## Captured Product Insight

This is a smoke-test level signal:

- SKAI should visually behave like a real LLM interface when rendering model output.
- "Cold-start" must be defined precisely. SKAI should avoid hidden provider memory, but it still intentionally materializes problem context.
- The user must be able to inspect exactly what SKAI added to the provider request.
- Problem context must be treated as background, not as an active instruction that overrides the user's latest prompt.
